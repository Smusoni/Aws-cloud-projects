What you’re practicing

  • S3 hosts your site files (HTML/CSS/JS) as objects.CloudFront caches them at edge locations for low latency.
  • Route 53 (optional) points a domain to your CloudFront distribution.

Deployment checklist

  1. Create an S3 bucket (unique name), disable “Block all public access”.
  2. Upload these files.
  3. Enable Static Website Hosting (index document: index.html).
  4. Add bucket policy for public read (s3:GetObject).
  5. Create a CloudFront distribution with your S3 website endpoint as the origin.

Pro move: after CloudFront is live, test caching by updating text and doing a hard refresh.
